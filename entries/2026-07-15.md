**Learned / clarified:**
- Vertex AI = GCP's UNIFIED ML platform (the umbrella; analog of AWS SageMaker). BigQuery ML is a TIER within the ladder, not the umbrella.
- The options ladder, least -> most custom effort: (1) Pre-trained APIs (call Google's model: Vision, Speech-to-Text, Translation), (2) BigQuery ML (train models in SQL on BigQuery data), (3) AutoML (low/no-code custom models on your data, in Vertex AI), (4) Custom training (your own code, full control). Principle: pick the lowest tier that solves the problem.
- Pre-trained API vs AutoML: generic solved task (e.g. transcribe calls -> Speech-to-Text API) uses a pre-trained API; a custom task on your own labeled data uses AutoML.
- AI infrastructure ladder: CPU (general) -> GPU (massively parallel) -> TPU (Google's custom ASIC purpose-built for matrix/tensor ops). AWS equivalents: TPU ~ Trainium (training) + Inferentia (inference); Vertex AI ~ SageMaker.
