**Learned:**
- Completed the GCP "Predict Visitor Purchases with BigQuery ML" module, including the hands-on lab (not just concept review). Rest week (R1), so kept it light: one module.
- The lab isn't really an SQL exercise; it's explicitly designed to have you prompt Gemini for the queries, not hand-write them. That was a bigger share of the workflow than expected this early in the GCP path.
- Ran two prompts against Gemini for the same underlying task and got two different (and non-equivalent) queries back:
  - Prompt 1 (loose, pasted the lab's task description almost verbatim: "predict the number of transactions each visitor makes, sort the results, select the top 10 visitors by transactions") returned a query that predicted per-row (per-session) `predicted_label` and sorted by that directly. Wrong shape: one visitor can have multiple rows/sessions, so this doesn't answer "top 10 visitors" at all.
  - Prompt 2 (specific: "predict the top 10 visitors by total predicted purchases using `bqml_lab.sample_model` and `bqml_lab.july_data`") returned the correct query, which aggregates with `SUM(predicted_label)` grouped by `fullVisitorId` before sorting/limiting.
  - The delta was a `GROUP BY` + `SUM()`, small syntactically, but it's the difference between "predicted value per session" and "predicted total per visitor." Vague prompting produced a query that runs cleanly and returns plausible-looking output while answering the wrong question.
- Started the Generative AI module. Foundation models are the backbone of gen AI, split into two categories:
  - Horizontal (large) foundation models: pretrained on massive, general data; used for conversation generation, information synthesis, content generation.
  - Vertical (small) models: fine-tuned on field-specific data for industries like retail, finance, healthcare; specialized problems.
- Three ways to build with these models: UI (no-code), APIs (low-code), SDKs (code, used via Colab/Workbench/Vertex AI).
