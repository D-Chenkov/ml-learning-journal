**Learned / reinforced:**
- OCI standardizes 3 specs: image-spec, runtime-spec, distribution-spec. runc implements the runtime-spec (low-level, creates the container); containerd is the high-level daemon that manages lifecycle and drives runc via the shim.
- Image layers are content-addressed and SHARED: a common base (python:3.12-slim) is stored once and reused across images. Tag = mutable, user-defined pointer (not a reliable version); digest (sha256) = immutable content hash = the real source of truth. Pin by digest for reproducibility.
- Multi-stage builds: a build stage does the heavy lifting (compilers/tools/source), then COPY --from=<stage> pulls only the built artifact into a slim final image. Keep what you need to RUN, drop what you needed to BUILD -> small image, small attack surface.
- A container lives exactly as long as its PID 1; when the main process exits, the container stops (so CMD must be the long-running foreground process). Restart policies (no / on-failure / always / unless-stopped) give automatic recovery from crashes; Compose uses `restart:`, K8s uses the pod restartPolicy + kubelet.
- Docker & AI (ch.10): Docker Model Runner runs LLMs on the HOST hardware (direct GPU/Metal access for performance), not inside a container; models are distributed as OCI artifacts (pulled from registries like images), exposes an OpenAI-compatible API, and integrates with Compose.
- Storage: Volumes (Docker-managed, persistent data like DBs - the default), Bind mounts (a specific host path, great for local dev / live source), tmpfs (RAM-only, no disk - for secrets and fast ephemeral scratch).
