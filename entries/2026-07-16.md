# 2026-07-16 - New research interest: ensemble / multi-agent consensus for trading

R1 "explore a topic that excites you" day. Logging a new self-directed research interest. No code yet and my notes are still rough, so this captures the interest + what I read, not a design.

**The interest:** ensemble learning for stock trading as a "debate" between specialized strategy agents (short-term day trader, long-term investor, pattern expert, ticker expert), each contributing to a consensus on what to trade, how much, and for how long, plus a "notetaker" that translates the debate into human-readable reasoning. Longer term I'd like to build a reusable "starter/template" for getting models to communicate and reach consensus - same spirit as k8s-fastapi-starter, but for ensemble/multi-agent orchestration.

**Read so far:**
- KL White Paper (sifsystemsmcrd.com) - a proposed "Kernel Language" for model-to-model communication.
- IEEE Xplore paper (doc 9175549).
- Renaissance Technologies business breakdown (danielscrivner.com).
- Kelly criterion (Wikipedia).

**Honest framing to carry (from discussing it):**
- Anchor the project on the ENGINEERING + RIGOROUS EVALUATION, not "beat the market." The credible, interview-worthy version is "a reusable ensemble-orchestration + backtesting framework, honestly evaluated, including where it fails."
- Be skeptical of the "Kernel Language" framing: the white paper is a single-source, self-published, non-peer-reviewed document with no evidence of real adoption. Don't stake credibility on it. Real adjacent research to cite instead: multi-agent LLM debate, mixture-of-experts, classical ensemble methods. Agents can just exchange structured JSON (signal/confidence/rationale) - no exotic language needed.
- Trading-ML reality: near-efficient/adversarial markets; most good-looking backtests are overfit. Guard look-ahead bias, survivorship bias, transaction costs/slippage; use walk-forward/out-of-sample testing; compare to a buy-and-hold baseline. Renaissance is an extreme closed outlier - "inspired by," not "replicate."
- Sizing: fractional Kelly (full Kelly is dangerous with an unknown edge). Research/paper-trading only; not financial advice.
